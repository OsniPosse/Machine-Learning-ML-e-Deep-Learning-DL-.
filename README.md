# Machine-Learning-ML-e-Deep-Learning-DL-.
Caderno Tematico no NotebookLM sobre Machine Learning (ML) e Deep Learning (DL).
Contexto e Objetivos
Este repositório foi criado como parte do Desafio de Projeto da DIO, com o objetivo de desenvolver um Caderno Temático no NotebookLM sobre Machine Learning (ML) e Deep Learning (DL).

Os objetivos deste estudo são:

Construir uma base sólida sobre os fundamentos de ML e DL.

Entender como modelos aprendem padrões a partir de dados.

Explorar arquiteturas modernas de redes neurais.

Criar um conjunto de prompts reutilizáveis para revisões futuras.

Documentar o processo de engenharia de prompts e as “cicatrizes” adquiridas ao interagir com a IA.

Respostas longas demais → resolvido com “resuma em até 5 tópicos”.

Confusão entre CNN e RNN → resolvido especificando “arquitetura CNN para visão computacional”.

Machine Learning — Visão Geral

ML é a área da IA que cria modelos capazes de aprender padrões.

Tipos principais:

Supervisionado — aprende com rótulos.

Não supervisionado — descobre padrões.

Reforço — aprende por tentativa e erro.

 Deep Learning — Visão Geral
Código
Entrada → Camadas Ocultas (muitas) → Saída
Subárea do ML baseada em redes neurais profundas.

Excelente para visão computacional, NLP e reconhecimento de voz.

Requer grandes volumes de dados e GPUs.

 Arquiteturas Importantes
CNN — Convolutional Neural Networks
Usadas para visão computacional.

RNN / LSTM — Redes Recorrentes
Usadas para sequências e séries temporais.

Transformers — Arquitetura Moderna
Base de modelos como ChatGPT, Gemini, LLaMA.

 4.2 Glossário Essencial
Overfitting — modelo aprende demais os dados de treino.

Underfitting — modelo aprende de menos.

Gradiente — direção de maior variação usada no treinamento.

Backpropagation — algoritmo que ajusta pesos.

Época — uma passada completa pelos dados.

Batch — subconjunto de dados usado por vez.

Tensor — estrutura multidimensional usada em DL.

Função de perda — mede erro do modelo.

Regularização — técnicas para evitar overfitting.
